## Функциональное тестирование Telegram-бота для записи на консультации

Данный репозиторий содержит примеры тестовой документации, разработанной в рамках вузовского проекта по тестированию Telegram-бота, предназначенного для записи студентов к преподавателям.



###  Цель проекта
Обеспечить качество работы Telegram-бота, проверить корректность логики записи, обработку ошибок и удобство интерфейса для конечного пользователя.

###  Что было выполнено в ходе проекта:
*   **Анализ требований:** Изучена логика взаимодействия пользователя с ботом.
*   **Проектирование тестов:** Разработаны сценарии проверок, покрывающие как позитивные, так и негативные кейсы.
*   **Создание документации:** Составлен чек-лист и детальные тест-кейсы.
*   **Поиск дефектов:** Выявлены и задокументированы баг-репорты с детальными шагами воспроизведения.

###  Использованные навыки и инструменты
*   **Telegram**: Использовался как основное окружение для тестирования продукта.
*   **Obsidia**: Использовался как инструмент ведения тестовой документации.
*   **Браузер**: Использовался для изучения логики работы ботов и вузовских требований.
*   **Тест-дизайн**: Разделение проверок на позитивные и негативные тесты.
*   **Работа с тестовой документацией**: Составление чек-листов, написание тест-кейсов с пошаговым алгоритмом и четким ожидаемым результатом.
*   **Документирование дефектов**:Оформление баг-репортов с разделением на фактический и ожидаемый результаты
